## 1. О протоколе NFS

**Network File System (NFS)** — протокол распределённой файловой системы, разработанный компанией Sun Microsystems в 1984 году. Он позволяет операционной системе обращаться к файлам, расположенным на другом компьютере сети, так же, как если бы они находились на локальном диске, делая физическое расположение хранилища прозрачным для приложений.

NFS работает по модели клиент/сервер: сервер **экспортирует** каталог и делает его доступным в сети; клиент **монтирует** его локально как собственный том. Обмен данными осуществляется посредством удалённых вызовов процедур (RPC) по протоколам TCP или UDP.

### Версии протокола

| Версия  | Год  | Ключевые особенности |
|---------|------|----------------------|
| NFSv2   | 1989 | Первая широко распространённая версия; только UDP; файлы до 2 ГБ |
| NFSv3   | 1995 | Поддержка TCP; большие файлы (64-битные смещения); асинхронная запись |
| NFSv4   | 2003 | Протокол с сохранением состояния; строгая аутентификация (Kerberos); единственный порт (2049); поддержка ACL |
| NFSv4.1 | 2010 | pNFS (параллельный доступ); устойчивые сессии; улучшенная поддержка кластеров |

### Типичные сценарии использования

- Общее хранилище между серверами приложений
- Централизованные домашние каталоги в Linux/Unix-окружениях
- Инфраструктура распределённого резервного копирования
- Общее хранилище для HPC-кластеров и Kubernetes
- Общий доступ к статическим ресурсам (изображения, видео) между веб-серверами

> **Безопасность:** NFS не шифрует передаваемые данные (кроме случаев использования Kerberos + RPCSEC_GSS). Рекомендуется всегда ограничивать доступ доверенными подсетями и никогда не открывать порты NFS в Интернет.

## 2. Функции плагина

- **Shares (NFS-экспорты):** определяет, какие локальные каталоги доступны удалённым клиентам, с контролем авторизованных IP (CIDR), режимом чтения/записи, синхронизацией и политикой сжатия прав доступа (squash).
- **Mounts (удалённые монтирования):** подключает общие ресурсы других NFS-серверов с полным набором параметров протокола: версия (NFSv3/NFSv4), размер блока (rsize/wsize), тайм-ауты, TCP/UDP, жёсткое/мягкое монтирование.
- **Мониторинг сервера:** состояние `nfs-server` и `rpcbind`, зарегистрированные RPC-сервисы, статистика протокола из `/proc/net/rpc`, метрики ввода-вывода на каждое монтирование через `nfsiostat`.
- **Автомонтирование при загрузке:** автоматически монтирует ресурсы, настроенные с `auto_mount=1`, через системный сервис инициализации.
- **Отслеживание соединений:** в режиме реального времени отслеживает активные монтирования и подключённых клиентов; обнаруживает изменения состояния вне плагина.
- **Интеллектуальная диагностика сбоев:** 7-шаговая последовательность проверок (ping, определение IP, проверка портов, showmount, сопоставление CIDR) с точной причиной и конкретными рекомендациями.
- **Журнал активности:** потокобезопасный журнал всех операций с фильтрами по типу события, результату и IP-адресу.

## 4. Установка

```bash
# 1. Скопировать плагин в директорию панели
cp -r nfs_free/ /www/server/panel/plugin/nfs_free/

# 2. Запустить установщик
sudo bash /www/server/panel/plugin/nfs_free/install.sh install
```

Установщик автоматически выполняет:

- Установку пакетов NFS операционной системы
- Фиксацию mountd на порту 20048
- Открытие портов брандмауэра в ufw (если активен)
- Регистрацию init-сервиса `/etc/init.d/nfs_free`
- Активацию `nfs-server` и `rpcbind` через systemd
- Перезагрузку aaPanel

**Удаление:**

```bash
sudo bash /www/server/panel/plugin/nfs_free/install.sh uninstall
```

> Системные пакеты NFS и файл `/etc/exports` **не изменяются** при удалении.

**Ручная установка зависимостей (если автоматический установщик не справился):**

```bash
# Debian/Ubuntu
sudo apt update && sudo apt install -y nfs-kernel-server nfs-common rpcbind

# RHEL/CentOS/Rocky/Alma
sudo yum install -y nfs-utils rpcbind
```

## 6. Порты и брандмауэр

NFS требует открытия следующих портов на **сервере**:

| Порт  | Служба   | Протокол | Описание |
|-------|----------|----------|----------|
| 111   | rpcbind  | TCP/UDP  | Распределитель портов — обнаруживает RPC-сервисы |
| 2049  | nfs      | TCP      | Основной протокол NFS |
| 20048 | mountd   | TCP/UDP  | Запросы монтирования (фиксированный порт) |
| 32874 | lockd    | TCP/UDP  | Блокировка файлов |
| 32876 | statd    | TCP/UDP  | Восстановление состояния |

По умолчанию mountd использует случайный порт, назначаемый rpcbind. Плагин настраивает `/etc/nfs.conf` для фиксации на порту **20048**, обеспечивая предсказуемые правила брандмауэра между серверами.

```bash
# Открыть порты для определённой подсети через ufw
sudo ufw allow from 192.168.1.0/24 to any port 111
sudo ufw allow from 192.168.1.0/24 to any port 2049
sudo ufw allow from 192.168.1.0/24 to any port 20048
```

## 8. Автомонтирование при загрузке

Сервис `/etc/init.d/nfs_free` автоматически монтирует все ресурсы с `auto_mount=1` при запуске системы.

```
Система загружается
  → /etc/init.d/nfs_free start
    → nfs_free_service (Python)
      → nfs_free_main.auto_mount()
        → для каждого монтирования с auto_mount=1:
            проверить, смонтировано ли уже (mountpoint -q)
            если нет: выполнить mount -t nfs -o ...
```

```bash
/etc/init.d/nfs_free start    # Запустить
/etc/init.d/nfs_free stop     # Остановить
/etc/init.d/nfs_free restart  # Перезапустить
/etc/init.d/nfs_free status   # Статус
```

## 9. Диагностика сбоев

При неудачном монтировании система автоматически выполняет 7-шаговую последовательность проверок:

| Шаг | Проверка | Инструмент |
|-----|---------|------------|
| 1   | Доступность сервера | `ping -c 1 -W 2` |
| 2   | Исходящий IP клиента | `ip route get` |
| 3   | Порты 111, 2049, 20048 | `/dev/tcp` |
| 4   | Список экспортов сервера | `showmount -e` |
| 5   | Наличие пути в экспортах | анализ вывода showmount |
| 6   | IP клиента в правилах доступа | `ipaddress.ip_network()` CIDR |
| 7   | Подсказки из сообщения об ошибке ядра | анализ ключевых слов |

## 11. Backend API

Класс `nfs_free_main` предоставляет 32 метода в 5 категориях:

| Категория | Основные методы |
|---|---|
| Экспорты | `get_share_list`, `create_share`, `modify_share`, `remove_share`, `show_ip_share_list` |
| Монтирования | `get_mount_list`, `create_mount`, `modify_mount`, `remove_mount`, `to_mount`, `to_umount`, `auto_mount` |
| Сервер | `get_server_status`, `server_admin`, `get_overview`, `get_nfsstat`, `get_nfsiostat` |
| Соединения | `get_connections`, `get_disk_mounts`, `get_nfs_ports`, `fix_mountd_port` |
| Журнал | `get_log`, `clear_log` |
